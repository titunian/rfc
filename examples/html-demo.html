The proposal

Three workstreams, sequenced so each unblocks the next.

  1. 01
    Content types
    Ship contentType: "html" | "markdown" on the plans table. Sanitize on render via DOMPurify. Bypass the markdown pipeline when html.
    SHIPPED
  2. 02
    Block model
    Convert both renderers to a shared block AST so comments anchor to stable IDs instead of text snippets.
    IN DESIGN
  3. 03
    Inline diff v2
    Replace line-LCS with token-aware diff that handles HTML attribute reordering. Behind ORFC_DIFF_V2, off by default.
    PLANNED

"The doc is the artifact. We've spent two years pretending it's the pull request — every time the doc lost it was because the PR didn't carry intent. HTML support is intent."

— internal review notes, 2026-04-30

Risks & mitigations

Risk Likelihood Mitigation
HTML XSS via missed sanitizer rule Medium DOMPurify allow-list test suite; CSP script-src 'self' as a second wall
Comment anchors break on re-rendered HTML High Block-AST migration in workstream 2; keep text-snippet fallback during the transition
Diff UX regressing on large HTML files Medium Token-aware diff behind a flag; perf budget < 80ms p95 on a 200KB doc
Bundle size from DOMPurify Low +8KB on /p/[slug] only — within the 250KB first-load budget
