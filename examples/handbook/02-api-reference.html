orfc handbook / API reference

API reference

Every endpoint orfc exposes. The CLI is built on these — there's nothing it can do that you can't reach with curl and a Bearer token.

Authentication

Web requests use a NextAuth session cookie. CLI requests use a Bearer token issued by orfc login: 

curl https://orfc.dev/api/plans -H "Authorization: Bearer orfc_live_…"

Plans

Method Path Description
GET /api/plans List the caller's plans. Optional ?folder= and ?tag= filters.
POST /api/plans Create a plan. Body accepts title, content, contentType, accessRule, folderPath, tags.
GET /api/plans/{id} Fetch one plan. Returns 401/403 if access rules don't match the caller.
PUT /api/plans/{id} Update a plan. Optimistic concurrency via expectedVersion. Old content is snapshotted into plan_versions.
DELETE /api/plans/{id} Delete a plan. Author-only. Cascades to comments and versions.
GET /api/plans/{id}/versions List previous versions for the version-history sidebar.
GET /api/plans/{id}/diff Line-LCS diff between two versions. Capped at 5,000 lines.

Comments & notifications

Method Path Description
GET /api/plans/{id}/comments List comments. Polled every 15s by the plan view.
POST /api/plans/{id}/comments Add a comment. Anchors via anchorText + block index.
PATCH /api/plans/{id}/comments/{cid} Toggle resolved. Plan author or comment author only.
POST /api/notify Email + Slack notify. URL must match app domain.

Example: publish an HTML plan 

curl https://orfc.dev/api/plans \
  -H "Authorization: Bearer $ORFC_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Q2 platform plan",
    "content": "<h1>orfc Q2 …</h1>",
    "contentType": "html",
    "accessRule": "anyone",
    "folderPath": "platform/2026",
    "tags": ["roadmap", "html"]
  }'

Status codes

2xx Success. JSON body.
401 No auth or expired session. Run orfc login.
403 Authed but not on the viewer list.
409 Concurrency. Re-fetch and retry.
410 Plan expired (used --expires).
500 Server error. Check Vercel logs.
